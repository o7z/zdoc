# EJS 代码块预览

zdoc 把所有 ` ```ejs ` 围栏代码块识别为**可交互模板**：读者打开页面后会看到一个内联面板，里面是自动生成的输入表单 + 实时渲染结果。作者只写模板本身，**不需要附带数据、不需要额外标记**。

## 最小例子

试试下面这块——直接改右侧表单字段，渲染结果会实时跟着变。

```ejs
<h1>Hello <%= name %></h1>
```

## 条件分支

`<% if (...) { %> ... <% } %>` 形式会被推断为布尔字段，表单出现 ☑ 勾选框。

```ejs
<% if (admin) { %><p>管理员视图</p><% } else { %><p>普通用户视图</p><% } %>
```

## 数字字段

只要模板里出现算术运算或数字比较（`>` / `<` / `+` 等），变量就会被推断为数字，表单出现数字输入框。

```ejs
当前年龄 <%= age %> 岁
<% if (age > 18) { %><p>已成年</p><% } %>
```

## 列表渲染

`for...of` 循环里的数组会被识别为**可增删的列表**，循环变量按内部 `.属性` 用法推断出元素 shape。

```ejs
<ul>
<% for (const item of items) { %>
  <li><%= item.name %> — <% if (item.active) { %>启用<% } else { %>停用<% } %></li>
<% } %>
</ul>
```

## 嵌套对象

属性链 `<%= user.profile.name %>` 会展开成多层折叠子表单。

```ejs
<div>
  <h2><%= user.profile.name %></h2>
  <p>城市：<%= user.profile.address.city %></p>
  <p>会员：<% if (user.settings.vip) { %>VIP<% } %></p>
</div>
```

## 嵌套列表

数组里嵌数组也支持。下面 `tags` 是 `items` 元素上的子数组。

```ejs
<% for (const item of items) { %>
  <h3><%= item.title %></h3>
  <ul>
  <% for (const tag of item.tags) { %>
    <li><%= tag %></li>
  <% } %>
  </ul>
<% } %>
```

## 不支持的情况

含**任何函数调用**的模板（自由函数、方法调用、EJS include 都算）会**静默关闭预览**——代码块仍正常显示语法高亮，只是不出现交互面板。

```ejs
今天是 <%= formatDate(today) %>
```

上面这块就不会出预览按钮，因为 `formatDate(...)` 是函数调用。zdoc 不会内置 helper、也不允许作者注入任意 JS——这是有意为之，保留了沙箱的安全边界。

### 处理办法

如果你需要预格式化数据，把它**在生成模板数据时就处理好**，模板里只保留纯展示逻辑：

```ejs
今天是 <%= todayFormatted %>
```

这样既能预览，也是 EJS 的最佳实践（模板不做计算）。

## 渲染环境

- **沙箱**：所有 EJS 编译和执行都在浏览器 Web Worker 里进行，1 秒超时强制 kill，runaway 循环不会冻结页面。
- **输出**：纯文本展示（即使输出是 HTML，也按源码字符显示），右上角带复制按钮。
- **重置**：表单顶部的"重置数据"按钮会把所有字段恢复成默认空值。
- **不持久化**：刷新页面表单数据会清空——这是文档站，不是 REPL。

## 与 lint 的配合

`zdoc lint` 会同时检查 ejs 块：

- 语法错误（编译失败）→ **error**，构建失败
- 空 ejs 块 → **warning**
- 同一变量既被当字符串又被当对象用（`<%= name %>` + `<%= name.first %>` 同时出现）→ **warning**，作者大概率写错
- 含函数调用 → **静默跳过**（不警告，因为预览自然也不会出按钮，作者意图清晰）

## 何时该用 ejs 预览

适合的场景：
- 介绍邮件 / 通知 / SQL / YAML / 配置 / 代码生成模板的文档
- 解释模板替换语法本身的教学材料
- 任何"输入数据 → 输出文本"的可视化演示

不适合的场景：
- 模板带复杂业务逻辑、依赖外部数据源——继续用传统的"贴模板 + 贴样例输出"二段式写法
- 一句话能说清的语法点——直接写 ` ```text ` 静态代码块就够了
